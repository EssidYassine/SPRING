📘 Chapitre 1 — Introduction à Spring Boot

Qu'est-ce que Spring Boot ?

Spring Boot est un framework Java basé sur Spring Framework qui simplifie la création d'applications standalone, prêtes pour la production. Il élimine la configuration XML manuelle grâce à l'auto-configuration.

Avantages de Spring Boot

  • Auto-configuration : configure automatiquement les beans selon les dépendances présentes
  • Starters : ensembles de dépendances pré-configurées (spring-boot-starter-web, spring-boot-starter-data-jpa...)
  • Serveur embarqué : Tomcat/Jetty intégré, pas besoin de déployer sur un serveur externe
  • Actuator : monitoring et métriques intégrés
  • Convention over configuration : moins de code boilerplate

@SpringBootApplication

@SpringBootApplication  // = @Configuration + @EnableAutoConfiguration + @ComponentScan
public class MonApplication {
    public static void main(String[] args) {
        SpringApplication.run(MonApplication.class, args);
    }
}

Associations JPA

// OneToMany — Un Projet a plusieurs Acteurs
@Entity
public class Projet {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String titre;

    @ManyToMany
    @JoinTable(
        name = "projet_acteur",
        joinColumns = @JoinColumn(name = "projet_id"),
        inverseJoinColumns = @JoinColumn(name = "acteur_id")
    )
    private List<Acteur> acteurs = new ArrayList<>();
}

@Entity
public class Acteur {
    @ManyToMany(mappedBy = "acteurs")
    private List<Projet> projets = new ArrayList<>();
}

Service — Opérations CRUD

@Service
@Transactional
public class ProjetService {
    @Autowired private ProjetRepository projetRepo;
    @Autowired private ActeurRepository acteurRepo;

    // Affecter un acteur à un projet
    public void affecter(Long projetId, Long acteurId) {
        Projet projet = projetRepo.findById(projetId).orElseThrow();
        Acteur acteur = acteurRepo.findById(acteurId).orElseThrow();
        projet.getActeurs().add(acteur);
        projetRepo.save(projet);
    }

    // Désaffecter un acteur d'un projet
    public void desaffecter(Long projetId, Long acteurId) {
        Projet projet = projetRepo.findById(projetId).orElseThrow();
        projet.getActeurs().removeIf(a -> a.getId().equals(acteurId));
        projetRepo.save(projet);
    }

    // Recherche par mot-clé
    public List<Projet> searchByKeyword(String keyword) {
        return projetRepo.findByTitreContaining(keyword);
    }
}

Controller REST

@RestController
@RequestMapping("/api/projets")
public class ProjetController {
    @Autowired private ProjetService service;

    @GetMapping
    public List<Projet> list() { return service.findAll(); }

    @PostMapping
    public Projet create(@RequestBody Projet p) { return service.save(p); }

    @PutMapping("/{projetId}/acteurs/{acteurId}")
    public void affecter(@PathVariable Long projetId, @PathVariable Long acteurId) {
        service.affecter(projetId, acteurId);
    }

    @DeleteMapping("/{projetId}/acteurs/{acteurId}")
    public void desaffecter(@PathVariable Long projetId, @PathVariable Long acteurId) {
        service.desaffecter(projetId, acteurId);
    }

    @GetMapping("/search")
    public List<Projet> search(@RequestParam String keyword) {
        return service.searchByKeyword(keyword);
    }
}

🎯 Chapitre 4 — Spring AOP (Aspect Oriented Programming)

Concepts clés

TermeDéfinition
AspectModule qui encapsule une préoccupation transversale (logging, sécurité...)
AdviceLe code à exécuter (Before, After, Around...)
PointcutExpression qui sélectionne les JoinPoints où s'applique l'advice
JoinPointPoint d'exécution du programme (appel de méthode)
WeavingProcessus qui lie l'aspect au code cible

Exemple d'Aspect — Logging

@Aspect
@Component
public class LoggingAspect {

    // Avant toute méthode dans le package services
    @Before("execution(* com.example.services.*.*(..))")
    public void beforeMethod(JoinPoint jp) {
        System.out.println("Appel : " + jp.getSignature().getName());
    }

    // Après exécution (succès ou exception)
    @After("execution(* com.example.services.*.*(..))")
    public void afterMethod(JoinPoint jp) {
        System.out.println("Fin : " + jp.getSignature().getName());
    }

    // Autour de la méthode (contrôle complet)
    @Around("execution(* com.example.services.*.*(..))")
    public Object aroundMethod(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        Object result = pjp.proceed();  // Appel de la méthode réelle
        long duration = System.currentTimeMillis() - start;
        System.out.println("Durée : " + duration + "ms");
        return result;
    }

    // Seulement si succès
    @AfterReturning(pointcut = "execution(* com.example.services.*.*(..))", returning = "result")
    public void afterSuccess(JoinPoint jp, Object result) { }

    // Seulement si exception
    @AfterThrowing(pointcut = "execution(* com.example.services.*.*(..))", throwing = "ex")
    public void afterError(JoinPoint jp, Exception ex) { }
}

Syntaxe Pointcut

"execution(* com.example.services.*.*(..))"
//           ^   ^^^^^^^^^^^^^^^^^^^^^^^  ^  ^^^
//           |   package.classe           |  args (..= tous)
//        retour                       méthode (*=toutes)
Pour activer AOP, ajouter dans pom.xml : spring-boot-starter-aop

📦 Chapitre 5 — DTO & Mapper

Qu'est-ce qu'un DTO ?

Un Data Transfer Object est un objet simple qui transporte des données entre les couches sans logique métier. Il découple l'API externe de la structure interne des entités.

Avantages des DTOs

  • Sécurité : ne pas exposer les champs sensibles de l'entité
  • Performance : envoyer seulement les données nécessaires
  • Découplage : l'API n'est pas liée au schéma de la BDD
  • Validation : appliquer des règles de validation à l'entrée
  • Évolution : modifier la BDD sans casser l'API

Exemple — Entité vs DTO

// Entité (avec mdp, données internes...)
@Entity
public class Acteur {
    private Long id;
    private String nom;
    private String motDePasse; // Ne pas exposer !
    private List<Projet> projets;
}

// DTO (seulement ce dont le client a besoin)
public class ActeurDTO {
    private Long id;
    private String nom;
    private String nationalite;
    // Getters, Setters, Constructeurs
}

Mapper Manuel

@Component
public class ActeurMapper {
    public ActeurDTO toDTO(Acteur acteur) {
        ActeurDTO dto = new ActeurDTO();
        dto.setId(acteur.getId());
        dto.setNom(acteur.getNom());
        dto.setNationalite(acteur.getNationalite());
        return dto;
    }

    public Acteur toEntity(ActeurDTO dto) {
        Acteur a = new Acteur();
        a.setNom(dto.getNom());
        a.setNationalite(dto.getNationalite());
        return a;
    }
}

ModelMapper

// Dans pom.xml :
// <dependency><groupId>org.modelmapper</groupId><artifactId>modelmapper</artifactId></dependency>

@Bean
public ModelMapper modelMapper() { return new ModelMapper(); }

// Utilisation dans le service :
@Autowired private ModelMapper mapper;

ActeurDTO dto = mapper.map(acteur, ActeurDTO.class);
Acteur entity = mapper.map(dto, Acteur.class);

Service avec DTO

@Service
public class ActeurService {
    @Autowired private ActeurRepository repo;
    @Autowired private ModelMapper mapper;

    public List<ActeurDTO> findAll() {
        return repo.findAll().stream()
            .map(a -> mapper.map(a, ActeurDTO.class))
            .collect(Collectors.toList());
    }

    public ActeurDTO save(ActeurDTO dto) {
        Acteur entity = mapper.map(dto, Acteur.class);
        return mapper.map(repo.save(entity), ActeurDTO.class);
    }
}

⏱️ Chapitre 6 — Spring Scheduler

Activer le Scheduler

@SpringBootApplication
@EnableScheduling  // OBLIGATOIRE pour activer les tâches planifiées
public class Application { ... }

Types de planification

AttributDescriptionExemple
fixedRateExécute toutes les N ms (depuis le début de la dernière)fixedRate = 5000
fixedDelayAttend N ms après la fin de la dernière exécutionfixedDelay = 3000
cronExpression cron (planification précise)cron = "0 0 8 * * ?"
initialDelayDélai avant la première exécutioninitialDelay = 1000

Expression Cron — Syntaxe

// Format : seconde minute heure jour-du-mois mois jour-de-semaine
//          0-59   0-59   0-23  1-31          1-12  0-7 (0=Dimanche)

"0 0 8 * * ?"     // Tous les jours à 8h00
"0 30 9 * * MON"  // Tous les lundis à 9h30
"0 0 0 1 * ?"     // Le 1er de chaque mois à minuit
"0 0/15 * * * ?"  // Toutes les 15 minutes
"0 0 8-18 * * ?"  // Toutes les heures de 8h à 18h

Exemples de Schedulers

@Component
public class MonScheduler {

    @Autowired private ProjetService projetService;

    // Tous les jours à 8h00
    @Scheduled(cron = "0 0 8 * * ?")
    public void envoyerRapportQuotidien() {
        List<Projet> projets = projetService.findAll();
        System.out.println("Rapport quotidien : " + projets.size() + " projets");
        // Logique d'envoi email, notification, etc.
    }

    // Toutes les 5 secondes
    @Scheduled(fixedRate = 5000)
    public void verifierEtat() {
        System.out.println("Vérification état : " + new Date());
    }

    // Attendre 3s après la fin de la tâche précédente
    @Scheduled(fixedDelay = 3000, initialDelay = 1000)
    public void nettoyage() {
        System.out.println("Nettoyage effectué");
    }
}
Sans @EnableScheduling, les méthodes @Scheduled ne s'exécutent PAS — erreur classique d'examen !

📖 Toutes les Définitions

Toutes les définitions importantes à connaître pour l'examen
🌿 Spring Boot
Framework Java basé sur Spring qui facilite la création d'applications standalone et prêtes pour la production, en éliminant la configuration manuelle grâce à l'auto-configuration et aux starters.
🔄 IoC (Inversion of Control)
Principe où le contrôle de la création et du cycle de vie des objets est délégué au framework (conteneur Spring) plutôt qu'au code de l'application. Au lieu que l'objet crée ses dépendances, c'est le conteneur qui les lui fournit.
💉 Injection de Dépendances (DI)
Mécanisme par lequel le conteneur Spring fournit automatiquement les dépendances d'un objet. Types : par constructeur (recommandé), par champ (@Autowired), par setter.
🫛 Bean Spring
Objet géré par le conteneur IoC de Spring. Un bean est instancié, configuré et géré par le conteneur. Créé avec @Component, @Service, @Repository, @Controller, ou @Bean dans une @Configuration.
⚙️ Auto-Configuration
Mécanisme Spring Boot qui configure automatiquement les beans selon les dépendances présentes dans le classpath. Ex : si spring-boot-starter-data-jpa est présent, Spring configure automatiquement EntityManager, DataSource, etc.
📦 Starter
Dépendance Maven/Gradle qui regroupe un ensemble cohérent de bibliothèques pré-configurées pour une fonctionnalité donnée. Ex : spring-boot-starter-web inclut Spring MVC + Tomcat + Jackson.
🗄️ JPA (Java Persistence API)
Spécification Java qui définit comment mapper des objets Java vers des tables de base de données relationnelles (ORM). Hibernate est l'implémentation la plus utilisée. Spring Data JPA simplifie encore plus son utilisation.
📋 ORM (Object-Relational Mapping)
Technique qui permet de manipuler une base de données relationnelle via des objets Java. Le framework ORM traduit les opérations sur les objets en requêtes SQL.
🏭 Repository
Interface Spring Data qui abstrait l'accès aux données. JpaRepository fournit des méthodes CRUD prêtes à l'emploi (findAll, findById, save, delete) et permet de définir des méthodes dérivées (findByNom, findByKeywordContaining...).
🔍 JPQL (Java Persistence Query Language)
Langage de requête orienté objet basé sur SQL mais qui opère sur les entités Java et leurs attributs (pas sur les tables/colonnes SQL). Utilisé avec @Query. Ex : "SELECT a FROM Acteur a WHERE a.nom = :nom" où Acteur est la classe Java.
📤 DTO (Data Transfer Object)
Objet simple (POJO) utilisé pour transporter des données entre les couches de l'application ou vers l'extérieur (API REST). Il ne contient pas de logique métier, seulement des données. Permet de découpler la structure interne de l'API publique.
🗺️ Mapper
Composant qui convertit une entité en DTO et vice versa. Peut être manuel (@Component avec méthodes de conversion) ou automatique (ModelMapper, MapStruct qui génère le code à la compilation).
🎯 AOP (Aspect Oriented Programming)
Paradigme de programmation qui permet de séparer les préoccupations transversales (logging, sécurité, transactions...) du code métier principal. En Spring AOP, un Aspect est appliqué autour des méthodes ciblées par un Pointcut.
🔪 Aspect
Module AOP qui encapsule une préoccupation transversale. Annoté avec @Aspect + @Component. Contient des Advices et des Pointcuts.
📍 Pointcut
Expression qui définit quels JoinPoints (méthodes) sont interceptés par un Advice. Syntaxe : execution(* com.example.services.*.*(..))
⚡ Advice
Action exécutée par un Aspect à un JoinPoint. Types : @Before (avant), @After (après, quoi qu'il arrive), @AfterReturning (après succès), @AfterThrowing (après exception), @Around (autour, contrôle total).
⏱️ Scheduler (Planificateur de tâches)
Mécanisme Spring pour exécuter automatiquement des méthodes à des moments précis ou à intervalles réguliers. Activé avec @EnableScheduling, les méthodes sont annotées @Scheduled avec des expressions cron ou des intervalles en ms.
📅 Expression Cron
Chaîne de 6 champs (sec min heure jour mois jour-semaine) qui définit quand exécuter une tâche. Ex : "0 0 8 * * ?" = tous les jours à 8h00. Le "?" signifie "non spécifié".
🔗 Association JPA
Relation entre entités JPA : @OneToOne, @OneToMany, @ManyToOne, @ManyToMany. Une association ManyToMany nécessite une table de jointure (@JoinTable). La gestion des associations (lazy/eager loading, cascade) est cruciale pour la performance.
🏗️ Maven
Outil de build et gestion de dépendances Java. Utilise un fichier pom.xml pour déclarer les dépendances, plugins et configuration du projet. Les dépendances Spring Boot sont gérées via le spring-boot-starter-parent qui fixe les versions compatibles.
🔄 @Transactional
Annotation qui délimite une transaction de base de données. Si une exception survient, toutes les opérations de la méthode sont annulées (rollback). Appliquée au niveau de la classe Service.
📡 REST (Representational State Transfer)
Architecture pour les API web basée sur HTTP. Utilise les verbes HTTP : GET (lire), POST (créer), PUT (modifier), DELETE (supprimer). @RestController = @Controller + @ResponseBody (retourne JSON directement).

Point 1
Affectation + Ajout
Affecter un acteur à un projet (ManyToMany) + Sauvegarder
// Service — affecter un acteur à un projet
@Transactional
public void affecterActeurAuProjet(Long projetId, Long acteurId) {
    Projet projet = projetRepo.findById(projetId)
        .orElseThrow(() -> new RuntimeException("Projet non trouvé"));
    Acteur acteur = acteurRepo.findById(acteurId)
        .orElseThrow(() -> new RuntimeException("Acteur non trouvé"));

    projet.getActeurs().add(acteur);  // Ajout dans la liste
    projetRepo.save(projet);           // OBLIGATOIRE pour persister
}

// Controller
@PutMapping("/{projetId}/acteurs/{acteurId}")
public ResponseEntity<Void> affecter(@PathVariable Long projetId, @PathVariable Long acteurId) {
    service.affecterActeurAuProjet(projetId, acteurId);
    return ResponseEntity.ok().build();
}

Point 4
JPQL — Requêtes personnalisées
Maîtriser la syntaxe JPQL avec @Query et @Param
// JPQL = SQL mais sur les CLASSES Java, pas les tables

// Sélection simple
@Query("SELECT a FROM Acteur a WHERE a.nationalite = :nat")
List<Acteur> findByNationalite(@Param("nat") String nat);

// LIKE (recherche partielle)
@Query("SELECT a FROM Acteur a WHERE a.nom LIKE %:keyword%")
List<Acteur> searchActeurs(@Param("keyword") String keyword);

// Jointure entre entités
@Query("SELECT p FROM Projet p JOIN p.acteurs a WHERE a.id = :acteurId")
List<Projet> findProjetsByActeur(@Param("acteurId") Long acteurId);

// Plusieurs conditions
@Query("SELECT a FROM Acteur a WHERE a.nationalite = :nat AND a.nom LIKE %:keyword%")
List<Acteur> searchByNatAndKeyword(@Param("nat") String nat, @Param("keyword") String keyword);
JPQL utilise les noms des classes Java (Acteur, Projet) et leurs attributs (nom, nationalite), PAS les noms des tables SQL !

Scheduler — Tous les jours à une heure précise + Implémentation
@EnableScheduling + @Scheduled(cron) + méthode qui appelle un service
// ÉTAPE 1 : Activer dans la classe principale
@SpringBootApplication
@EnableScheduling  // ← NE PAS OUBLIER
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// ÉTAPE 2 : Créer le composant Scheduler
@Component
public class ProjetScheduler {

    @Autowired
    private ProjetService projetService;

    // Tous les jours à 8h00 du matin
    @Scheduled(cron = "0 0 8 * * ?")
    public void tacheQuotidienne() {
        System.out.println("Tâche exécutée à : " + new Date());
        List<Projet> projets = projetService.findAll();
        System.out.println("Nombre de projets : " + projets.size());
        // Logique métier : envoi rapport, nettoyage, notification...
    }

    // Tous les jours à 23h30
    @Scheduled(cron = "0 30 23 * * ?")
    public void nettoyageQuotidien() {
        System.out.println("Nettoyage nocturne lancé...");
    }
}

// Aide-mémoire Cron :
// "0  0  8  *  *  ?" → sec=0 min=0 heure=8 tousLesjours
// "0  0  9  *  *  MON" → Chaque lundi à 9h
// "0  */30  *  *  *  ?" → Toutes les 30 minutes
Format cron : sec min heure jour mois jour-semaine — "0 0 8 * * ?" = chaque jour à 8h00
Point 6
Désaffectation — Retirer une association
Supprimer un acteur d'un projet (removeIf) + save()
// Service — désaffecter un acteur d'un projet
@Transactional
public void desaffecterActeurDuProjet(Long projetId, Long acteurId) {
    Projet projet = projetRepo.findById(projetId)
        .orElseThrow(() -> new RuntimeException("Projet introuvable"));

    // Retirer l'acteur de la liste
    projet.getActeurs().removeIf(a -> a.getId().equals(acteurId));

    // Sauvegarder
    projetRepo.save(projet);
}

// Controller
@DeleteMapping("/{projetId}/acteurs/{acteurId}")
public ResponseEntity<Void> desaffecter(
        @PathVariable Long projetId,
        @PathVariable Long acteurId) {
    service.desaffecterActeurDuProjet(projetId, acteurId);
    return ResponseEntity.noContent().build();
}

📖 Définitions à Connaître par Cœur

Maven

Outil de build et gestion de dépendances Java. Le pom.xml déclare les dépendances (starters Spring Boot). spring-boot-starter-parent gère les versions compatibles automatiquement.

DTO (Data Transfer Object)

Objet simple qui transporte des données entre couches ou vers l'API. Découple la structure de la BDD de l'API, sécurise (cache les champs sensibles), améliore les performances.

IoC (Inversion of Control)

Le conteneur Spring crée et gère les objets (beans) à la place du développeur. Le code ne crée plus ses propres dépendances avec new.

@Autowired

Spring injecte automatiquement la dépendance correspondante (bean du même type) dans le champ ou constructeur annoté.

JPQL

Langage de requête JPA orienté objet. Utilise les noms des classes Java et attributs (pas SQL). Utilisé avec @Query dans les repositories.

@Transactional

Délimite une transaction. Si une erreur survient, toutes les modifications de la méthode sont annulées (rollback). Obligatoire pour les opérations d'écriture complexes.
